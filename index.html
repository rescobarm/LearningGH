Agregando cambio !!

  • git init (Inicializa el repositorio)
  • git status (Status de los archivos)
  • git add -A ó git add filename (Agrega todos los archivos al stage área)
  • git rm --cached .gitignore (Saca de la lista del stage área)
  • git commit -m "Mensaje" (Agrega archvis al repositorio los archivos)
  • git log (Muestra commits realizados)
  • git diff (Muestra las diferencias que hay entre lo que estaba y los cambios realizados)
  • git pull origin master (Descargamos la última versión del códio del proyecto)

WORK FLOW!!! branch

  • 1.- git branch [branch name] (Crear rama para modificaciones)
  • 2.- get checkout [branch name] (Cambiar a la nueva rama)
  • 2.1.- Realizar cambios al código...
  • 3.- git add -A (Agrega los cambios realizados de la rama)
  • 4.- git commit -m "Confirmando cambios realizados en la nueva rama"
  • 5.- git push -u origin [branch name] (Indicar al repositorio master que existe una rama ligada par ser agregado)
  • 5.1.- git branch -a (Muestra todas las branch, remotas y locales)
  • 6.- get checkout master (Pasarnos a la rama master para poder unir los cambios a la rama principal)
  • 7.- git pull origin master (Verificar que se cuenta con la nueva versión del proyecto)
  • 8.- git branch --merge (Muestra las ramas que se han unidos)
  • 9.- git merge [branch name] (Une la rama al proyecto master)
